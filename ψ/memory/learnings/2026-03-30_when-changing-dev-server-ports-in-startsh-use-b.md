# When changing dev server ports in start.sh, use `bunx next dev -p PORT --turbo` 

When changing dev server ports in start.sh, use `bunx next dev -p PORT --turbo` directly instead of relying on package.json scripts with hardcoded ports. The bunx approach bypasses package.json configuration.
