# Session: Docker Oracle Studio Combined Deployment

## Session Summary

Successfully deployed Oracle Studio (dashboard) + Arra Oracle MCP as a single Docker container with:
- MCP HTTP on port 47778 (for Claude Code)
- Dashboard on port 47780 (Oracle Studio web UI)
- Internal REST API on port 47779 (dashboard → API)
- Vector embeddings via Ollama on Windows host

## What Got Done

### Code Changes
- Created `src/index-combined.ts` in arra-oracle-v2 — runs MCP + REST API together
- Modified `src/indexer/index.ts` to use `getVectorStoreByModel('bge-m3')` for correct collection naming
- Created `oracle-studio/Dockerfile` — combined build for both services
- Created `oracle-studio/docker-compose.yml` — orchestrated deployment with volume mounts
- Created `oracle-studio/docker-entrypoint.sh` — startup script for all services

### Configuration

- Volume mounts for ψ/ knowledge directory and persistent data

### Deployment
- 107 documents indexed with vector embeddings
- 3D knowledge map populated with 8 clustered points
- All services healthy and running

## Lessons Learned

1. **Docker Desktop networking quirk**: `host.docker.internal` resolves correctly for some calls but not all; `extra_hosts` with `host-gateway` is more reliable for WSL2→Windows Ollama

2. **Collection naming mismatch**: Indexer defaulted to `oracle_knowledge` but dashboard expected `oracle_knowledge_bge_m3` — use `getVectorStoreByModel()` to ensure consistency

3. **Ollama must listen on 0.0.0.0**: Windows Ollama needs `setx OLLAMA_HOST "0.0.0.0"` and restart to accept connections from WSL2/Docker

4. **Production deps exclude lancedb**: `bun install --production` doesn't include lancedb for eval scripts; need to run from arra-oracle-v2 directory

## Next Steps

1. Monitor for any Docker networking issues after sleep/restart
2. Consider adding healthcheck for vector store connectivity
3. Document update procedure for preserving data across upgrades
4. Add auto-reindex on container start when ψ/ changes detected
