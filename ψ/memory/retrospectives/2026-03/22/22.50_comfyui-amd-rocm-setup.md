## What Could Improve

1. **AMD AI Bundle quality**: It's genuinely broken for AMD GPUs - needs complete overhaul
2. **comfy_aimdo coupling**: Should be optional, not hard-imported everywhere
3. **Documentation**: AMD-specific setup guides are scattered and often outdated

## Honest Feedback

**Friction Point 1**: The AMD AI Bundle is misleading - it claims AMD support but ships NVIDIA-only code and CPU PyTorch. Users expect "it just works" but get hours of debugging.

**Friction Point 2**: comfy_aimdo is deeply embedded in ComfyUI's codebase with imports in main.py, execution.py, windows.py, ops.py, utils.py, pinned_memory.py, and model_patcher.py. This should be a soft dependency.

**Friction Point 3**: I kept trying to patch the broken AMD AI Bundle installation when the user was willing to do a fresh install earlier. Should have asked about nuclear option sooner.

## Lessons Learned

1. **Native ROCm > DirectML**: The performance difference is massive (110GB vs 1GB VRAM, actual GPU acceleration)
2. **gfx1151 is new**: Requires specific environment variables and bf16-unet (not plain bf16)
3. **Stub modules work**: Creating fake comfy_aimdo modules is cleaner than patching multiple files
4. **Ask about fresh install earlier**: Could have saved 2+ hours of patching broken installations
