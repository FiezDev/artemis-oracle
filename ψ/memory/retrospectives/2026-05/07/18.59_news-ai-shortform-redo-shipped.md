## Lessons Learned

- **The QOne workflow already had the right rules but they were bypassable.** The Codex `news-ai` session generated captions via local Python scripts (`scripts/aiinspire_*`) that never went through `nova-factcheck-expand` / `nova-translate-polish`, so the prompt-level rules ("never mention source video") never ran. **Fix**: hard-required output schema fields + a deterministic regex `nova-content-guard` step before the human gate, so a bypass attempt is caught at publish time even if the producer is bypassed entirely.

- **FB anti-automation degrades non-monotonically.** A fresh agent-browser session schedules ~8 posts cleanly, then starts failing — first on "Scheduling options not found" (UI element not rendered), then on `Navigation failed: net::ERR_ABORTED` (daemon-level). `--close-between-posts` cools this down because it forces a fresh page load + brief idle per post. Login itself stays good through this; it's the schedule UI that gets throttled.

- **2FA push approval has a tight FB window.** When `agent-browser open` reloads the FB page, FB's polling for the approval status restarts, and any prior approval attempt is invalidated. Headed mode + user completing the login by hand is the only reliable path past 2FA when the vault has only email+password.

- **Inline Thai regex needs no `\b`.** `re.compile(r"ในคลิป\b")` doesn't match because Thai script has no word boundaries. Drop `\b` for Thai patterns; rely on the literal token. Caused the first sanitization pass to leave 4 captions dirty.

- **Subagent JSON keys diverge from caller assumptions silently.** I sent the rewrite subagent a brief keyed by `it["index"]` (queue position 0-14), but the subagent inferred topic numbers (1-20) from filenames — both look like int strings, so the file shape didn't error. Validation caught it via "missing-อ้างอิง-line" violations, not a key mismatch. **Fix**: when the input data has two int-like identifiers, name them disambiguously (e.g., `queuePosition` vs `topicNumber`) in the brief.

- **"Approved" doesn't mean "ready to commit".** User approved the sanitized sheet, then approved the rewritten sheet — twice. Build the review artifact first, get explicit human sign-off, then mutate the live queue. Never post-and-rollback with FB.
