## Lessons Learned
- `clother-zai` IS Claude Code under the hood (`/home/bjgdr/.local/bin/clother-zai` resolves to a Claude wrapper). `--model glm-5.1` routes generation to GLM-5.1; `--yolo` is the alias for `--dangerously-skip-permissions`. Same tool surface (Read/Bash/Edit), different model.
- For controlled spawning, the pattern is `clother-zai --model glm-5.1 --yolo -p "<prompt>"` — `-p` gives one-shot non-interactive output to stdout. For long tasks, run in background and read stdout when done; do not poll.
- When the spawned agent's stdout contains tail noise (system-reminders, etc.), the actual answer is reliably on the last useful line — `tail -10` is sufficient for short answers; for richer output prefer JSON output mode (`--output-format json`) instead of parsing prose.
- "Test that you can control it" means two probes, not one: a literal-string probe proves obedience to output format, a tool-using probe proves real work capability. Single ping is not sufficient verification.
