## Lessons Learned

- Claude Code rate limits JSON uses `rate_limits.five_hour`, `rate_limits.seven_day`, `rate_limits.seven_day_opus` with `used_percentage` and `resets_at` fields
- `resets_at` is a Unix epoch (seconds), not ISO8601 — needs `date +%s` comparison
- Weekly limit data only appears after first API response in session — `n/a` before that is expected, not a bug
- Opus weekly sub-limit only present on Max plans
