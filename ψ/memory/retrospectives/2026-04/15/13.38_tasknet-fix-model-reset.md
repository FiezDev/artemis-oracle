## Lessons Learned

1. **Silent failures are the worst failure mode.** The `.catch(() => null)` pattern in relay.ts hid every API failure — port issues, missing endpoints, format mismatches — all invisible. Error logging is not optional for data pipelines.
2. **Data format mismatches cause silent zero-values.** When a stats endpoint returns arrays and consumers read objects, you get `undefined` everywhere — not errors, just wrong data. Always verify the shape of API responses, not just the status code.
3. **Config in Docker images requires container restarts.** The TaskNet API runs in Docker (`arra-oracle-unified`) with code baked in, not volume-mounted. Had to `docker cp` changed files and restart the process. Should consider volume-mounting the source for faster iteration.
