## Lessons Learned

1. **Container code sync is fragile.** The Docker container (`arra-oracle-unified`) has its own copy of the API code at `/app/tasknet/`. Local file changes don't take effect until manually `docker cp`-ed in and container restarted. The `schema/index.ts` overwrite broke things because it dropped the `comments.js` export that existed only in the container.

2. **Don't blindly overwrite shared files.** When copying files between host and container, the host version may be behind (missing exports, different imports). Always diff before overwriting.

3. **Supervisord manages services inside the container.** Can't just `kill` and restart — need full container restart for clean state, or supervisorctl access.

4. **Rate limits hit when spawning 4+ agents in parallel.** Background agents for UI, Event Bridge, and agent files all got 429'd. Had to do the work directly in main session.
