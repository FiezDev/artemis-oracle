## Lessons Learned

1. **Team removal is viral** — `team_id` was embedded in 6 database tables, 15+ route files, types, services, mock data, schemas, and migrations. A simple "remove team" request touched nearly every file. Lesson: shared concepts like ownership scoping spread everywhere.

2. **Parallel agents for bulk edits** — launching 2-3 agents to handle different file groups (types+mock-data vs backend-routes) was far more efficient than sequential edits. But each agent needs very precise instructions with file paths and line numbers.

3. **Workflow state needs JSON step_data** — rather than creating columns for every possible step output, the `step_data TEXT DEFAULT '{}'` pattern gives flexibility to store arbitrary per-step results (selected item, generated storyboard, clips) without schema changes.

4. **Build early, build often** — running `turbo build` after the sidebar restructure confirmed everything compiled before moving to the bigger page rewrites. Caught zero errors because we verified incrementally.

5. **Agent reliability varies** — one of three parallel agents hit a 400 network error. The retry succeeded. Don't assume all agents complete — check results and retry failures.
