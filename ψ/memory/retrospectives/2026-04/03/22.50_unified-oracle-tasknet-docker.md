## Lessons Learned

1. **TimescaleDB is Alpine-based** — `apt-get` doesn't work, must use `apk` or multi-stage build with `oven/bun:1`
2. **Docker Compose build context paths** — absolute paths work on single machine but aren't portable. External build contexts (`/home/bjgdr/...`) mean the compose file isn't self-contained
3. **Port conflicts on restart** — Old containers hold ports. Must `docker stop/rm` before `docker compose up` if ports overlap
4. **Oracle data lives in Docker volumes, not host SQLite** — The `oracle-studio` container has 173 indexed docs in its Docker volume. The host SQLite at `~/.arra-oracle-v2/oracle.db` only had 2 rows (settings + indexing_status)
5. **Bun MCP transport needs Node HTTP** — StreamableHTTPServerTransport requires `http.createServer()`, so TaskNet MCP runs on a separate Node server alongside the Hono/Bun API
