## Lessons Learned
- **reCAPTCHA false-positives come from the page's HTML, not its rendered text.** Early checkpoint detection matched the string `recaptcha` anywhere in `document.documentElement.outerHTML`, which firecrackered on healthy logged-in FB pages because FB loads a reCAPTCHA JS bundle preemptively. Narrowing the detector to (a) `document.body.innerText` and (b) iframe `src` containing a real reCAPTCHA widget path cleared the false positives. General principle: detect hostile states by what the user would *see*, not what the page has *loaded*.
- **Synthetic `el.click()` on React-bound buttons is silently dropped.** FB's post composer needs a trusted CDP click. The workaround that worked reliably across all DOM pokes: mark the target with `data-ab-*="1"`, then have `agent-browser click '[data-ab-*="1"]'` do a real CDP click. Keeping all identifier attributes under a common prefix made cleanup easy and debugging obvious.
- **Page-mode on Facebook lives on `web.facebook.com` home, not the page URL.** The `/profile.php?id=<pageId>` route only *selects* the page identity (via the "Switch Now" admin control). Opening the composer while in page mode from `/` posts as that page. Business Suite direct-composer URL returns "content not available" for pages not linked to a Business Manager — useless for personal AI-Inspired-style pages.
- **Drizzle's `inArray(col, [singleId])` generates `col IN $1` which Postgres rejects.** Use `eq(col, singleId)` when the list has exactly one element. This ate a script run before the fix landed.
- **Workflow-runs don't cascade on workflow delete.** Needed explicit ordered deletion: `reviewItems → topicDrafts → workflowStepRuns → workflowRuns → workflowSchedules → workflows`, then orphaned-gate cleanup afterwards. The schema enforces referential integrity exactly where it should.
- **`clother-zai mcp add` stdio flag order matters.** The variadic `-e <env...>` parser will happily eat the positional `<name>` if the name follows the `-e` list. Put the server name *before* the `-e` flags, then `--` then the command. This cost me one bad `zai-vision` registration before I saw it.
- **Next.js 14 `params` is a plain object; `use(params)` on it throws.** Next 15+ makes params a Promise. Check the `"next"` entry in package.json before copying a pattern from the docs.
- **Business Suite dialogs compose in layers.** After a successful post, FB often stacks a "Add WhatsApp button" upsell on top of the composer — the post DID go through, but the waiting-for-dialog-close loop sees the upsell and thinks the publish failed. Iterate *all* `[role=dialog]` elements and dismiss ones matching upsell text; don't assume the first dialog is the composer.
- **The split producer/publisher pattern is the right shape for scheduled content pipelines.** Producer runs daily and stacks a backlog; publisher consumes 1 per tick with `FOR UPDATE SKIP LOCKED`. Decoupled cadence, resilient to gate delays, trivially scalable to more platforms. Do this again next time.
- **Checkpoint-aware browser automation requires `--headed` + a persistent profile.** Headless sessions get hammered by reCAPTCHA Enterprise; headed + cookies-on-disk = FB trusts the device after one manual login. `--profile /home/bjgdr/.agent-browser/fb-profile` bakes the trust.
