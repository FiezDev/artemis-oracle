## What Got Done

### Bugs Fixed
1. **"Untitled task" in Telegram** — `blockTask()` didn't pass `taskTitle` in return value or `notifyTaskEvent`. Fixed across `blockTask`, `releaseTask` (2 paths), `completeTask`, `unblockTask`.
2. **"Summary unavailable" in 15m reports** — Concurrent OpenClaw sessions for same agent. Added per-agent serialization lock in relay.
3. **"Heartbeat 480m ago"** — Agent runners never called heartbeat endpoint. Added `sendHeartbeat()` with 5min interval to agent-runner.ts.
4. **Dead SQL in claimTask** — `higherPriority` query ran wrong SQL and discarded result. Removed.
5. **Stale reaper / stuck-agent race** — Both iterate `in_progress` tasks with no coordination. Added shared `underInvestigation` Set.
6. **Zombie guard activeCount drift** — `releaseTask` zombie path reset task to available but never decremented agent `activeCount`.
7. **No priority validation** — `updateTask` and `updateGoal` accepted invalid priority values.
8. **ReassignTask missing agent check** — Would fail at DB level with unhelpful error. Added existence validation.

## Lessons Learned

1. **Docker code is baked, not mounted** — After editing API source, must `docker compose build api && docker compose up -d api` to deploy. Changes to host files don't reach the container without rebuild.
2. **Edit tool whitespace sensitivity** — Tab vs space indentation and Unicode characters (em-dashes) cause silent match failures. Always `cat -A` to check exact whitespace before editing.
3. **Systemd user services manage everything** — Relay, agent runners, and OpenClaw gateway all run as `systemctl --user` services, not Docker. Don't confuse the two stacks.
4. **Scope creep in task naming** — "Docker hardening" bundled arra-oracle-unified changes with dashboard changes, causing confusion. Keep tasks scoped to one system.
