## Lessons Learned

1. **WSL2 Docker networking**: `host.docker.internal` resolves to Docker Desktop's VM gateway (192.168.65.254), NOT the WSL2 host. Only ports exposed by other Docker containers are forwarded. Host-only services (like systemd relay) need the eth0 IP directly.

2. **Silent failures compound**: The notification-service had TWO bugs stacked — wrong URL path AND unreachable host. Each would have been caught instantly with a single test POST, but because the service silently catches errors, both accumulated undetected.

3. **Data migration ordering matters**: pg_dump with `--clean` drops and recreates tables, but FK constraints cause COPY failures if tables restore in wrong order. `--data-only --disable-triggers` after a manual TRUNCATE is the clean path.

4. **Sprint exhaustion looks like broken infrastructure**: "Why aren't agents reporting?" had a simple answer — all 145 tasks were done. But it took tracing through runners, queues, and notification paths to confirm nothing was actually broken.
