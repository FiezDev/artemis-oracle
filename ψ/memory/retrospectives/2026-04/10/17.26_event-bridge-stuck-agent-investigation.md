## Lessons Learned

1. **Systemd services don't inherit shell environment** — nvm changes only affect the current shell. Must explicitly set PATH in the systemd service file OR pass env in Bun.spawn.
2. **sendDM sends to CEO, not the agent** — the function uses the agent's bot token to message CEO_TELEGRAM_ID. For waking agents, need to use openclaw CLI sessions instead.
3. **"Process crashed" is not a root cause** — the user wants to know WHY. System-level evidence (dmesg, journalctl, memory, disk) is needed to find the real reason.
4. **OpenClaw session files are JSONL** — at `~/.openclaw/agents/AGENT/sessions/` — can be read non-intrusively for analysis without triggering agent sessions.
5. **Docker PostgreSQL credentials** — user=qone, db=qone_workflows, password=qone123 for the arra-oracle-unified container.
