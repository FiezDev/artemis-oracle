# Session Retrospective — Created df-deep-build skill

## Session Summary
Combined two existing skills (df-refine-prompt and feature-dev:feature-dev) into a new unified skill called df-deep-build. The skill interviews the human through multiple rounds until requirements are crystal clear, then either compiles a mega-prompt (non-code tasks) or implements features with full codebase exploration, architecture design, and review (code tasks). Tested with 3 parallel subagents, identified 4 issues, and fixed them.

## Lessons Learned
- Skill auto-detection for code vs non-code tasks works reliably with keyword signals (tech names, implementation verbs, file paths)
- The COSTAR framework's "Examples" section needs special handling for code tasks — code examples are reference implementations or API request/response samples, not traditional input/output pairs
- Testing skills with parallel subagents is effective but can't replicate the human-gate behavior (subagents simulate answers rather than waiting for real approval)
- The completeness checklist pattern (✅/⚠️/❓) gives the human clear visibility into what's covered and what's still unknown, making the interview feel productive rather than endless
