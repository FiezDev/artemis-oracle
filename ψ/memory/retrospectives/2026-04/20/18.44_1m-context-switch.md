## Lessons Learned
- The `[1m]` model suffix is the cleanest lever for extended context — no separate beta header config or ANTHROPIC_BETAS juggling needed in Claude Code
- Opus 4.6 is Bedrock/Vertex/Foundry only; Anthropic API defaults to 4.7 — so "opus" in Claude Code = 4.7 unless you target another provider
- Knex "Failed query" prefix alone is diagnostically useless — the real error (ER_BAD_FIELD / ER_NO_SUCH_TABLE / connection) always lives on the next line. Always ask for it before guessing
- The login query listed `status_id` as the last SELECT column — a typical "newest migration not run locally" smell, but I held off proposing that until evidence arrives
