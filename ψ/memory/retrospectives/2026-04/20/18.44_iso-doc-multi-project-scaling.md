## Lessons Learned
- **Multi-repo system → one config → one doc set.** NTTAG (3 repos) and RiceGuard (10+ repos) and QMS (7 repos) all resolve to a single `iso-doc.json` with a `frontends[]` array. Router type is a frontend-level decision: `vue`, `static`, or Next `app-router`. One system = one set of docs, period.
- **When a URL list is the source of truth, stop parsing.** QMS and RiceGuard have URL prefixes served from different repos by an Nginx aggregator. Parsing each repo's router is fighting the deployment topology. `router_type: "static"` is the right abstraction — the URL list IS the contract.
- **OTP relay over stdin works when orchestrated interactively.** Pre-login outside capture via agent-browser + manual relay + `--skip-login` is cleaner than wiring stdin piping through the skill. Agent-browser's persistent Chrome profile carries cookies across hostnames (same eTLD+1).
- **Mermaid gotchas that cost time**: semicolons inside sequenceDiagram message text, and dots inside edge labels (`.m3u8`). Both break the lexer. Replace with simpler words; keep literal URLs out of diagram edge labels.
- **Undeployed frontends with HTML-fallback assets.** Cloudflare returning `text/html` for a `.js` asset path is the tell that a SPA is broken, not slow. `curl -I` the asset URL before attempting capture; drop from scope if 404/fallback.
- **Bilingual intersection with single-lang hosts.** Setting `i18n_mode: "none"` on a single-language admin frontend lets the per-lang capture loop run twice but both writes produce identical shots — both `en/` and `th/` dirs end up populated, intersection rule includes the routes cleanly.
