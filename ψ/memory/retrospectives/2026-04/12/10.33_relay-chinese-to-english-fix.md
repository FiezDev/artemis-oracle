## Session Summary

Fiez reported that Artemis relay messages in QOne were coming through in Chinese instead of English. Traced the root cause through the full stack and applied fixes.

## What Got Done

- **Investigated** the QOne Event Bridge relay system at `/home/bjgdr/dev-personal/qone_corp/event-bridge/`
- **Root cause identified:** GLM-5.1 (Zhipu AI) defaults to Chinese without explicit English enforcement. The prompt templates had no language instruction.
- **Fixed 4 prompts in 2 files:**
  - `prompts.ts` — Added `IMPORTANT: You MUST write your response in English only` to the shared `rules` string (covers all event types: created/claimed/completed/blocked/etc.) and the wake-up DM prompt
  - `relay.ts` — Added English-only instruction to the 15-minute progress report prompt and the stuck agent investigation prompt
- **Also discovered:** `/home/bjgdr/oracle/qone_company/skills/improvement/advanced-skill-creator/SKILL.md` contains Chinese trigger phrases (`写一个触发`, `创建技能`) — potential context contamination source but not directly causing relay issue

**No git commits** — changes are on filesystem at `/home/bjgdr/dev-personal/qone_corp/event-bridge/`, not in this repo.

## Lessons Learned

1. **Chinese AI models need explicit language enforcement.** GLM-5.1 will drift to Chinese unless the prompt says "English only" prominently. A single-line instruction isn't enough — use `IMPORTANT:` prefix and repeat at the end.
2. **Prompt templates are a single point of failure for language.** The `buildPrompt()` function's `rules` string is shared across all event types — one fix there covered 7+ message types.
3. **The QOne codebase lives in two places** — `/home/bjgdr/oracle/qone_company/` (Oracle config, skills, agent definitions) and `/home/bjgdr/dev-personal/qone_corp/` (actual runtime code: event-bridge, agents-v2, dashboard). Need to check both when debugging.
